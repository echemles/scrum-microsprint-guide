Terms in Plain Language

        

          
🎯
Sprint Goal
The one thing this hour is for.

          
📋
Sprint Backlog
The tiny list you commit to.

          
✅
Definition of Done
The checklist that prevents fake-done.

          
📦
Increment
The usable thing produced.

          
🔄
Retro
The improvement loop.

        


        
Two-Cofounder Model

        

          
Role A: Builder
  • Owns the sprint goal
  • Writes code / creates deliverables
  • Demos the increment

          
Role B: Scrum Master
  • Keeps time + scope
  • Asks facilitator questions
  • Runs review and retro

        

        
Switch roles each microsprint.


        
Scrum Master Script

        

          
Plan
  • What is the one sprint goal?
  • What 1-3 tasks will get us there?
  • What does done look like?

          
Build
  • Still on track?
  • Scope growing? Cut something.
  • Can you demo this yet?

          
Review
  • Show me the working thing.
  • Does it match the goal?
  • What feedback changes next?

          
Retro
  • What went well?
  • What slowed you down?
  • One change for next time?

        


        
Common Failure Modes

        

          
Too much scope — If you can't demo it in the last 10 min, you scoped too big.

          
Skipping review — No review = no verified output.

          
Skipping retro — You'll repeat the same mistakes.

          
Activity ≠ output — Lines of code ≠ shipped increment.

          
One person dominates — Both founders must own a deliverable.
