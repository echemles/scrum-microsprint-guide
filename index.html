Operating Model

Two-Cofounder Operating Model

Each founder owns their product work. The other acts as scrum master. Switch every sprint.

Role A: Builder
  • Owns the sprint goal for their project
  • Writes code / creates deliverables
  • Demos the increment at review
  • Reports blockers honestly
Role B: Scrum Master
  • Keeps the sprint on track (time + scope)
  • Asks the facilitator questions at each phase
  • Flags scope creep early
  • Runs the review and retro
Switch roles each microsprint. Both founders must own a deliverable.

Plan (0–5 min)

  • What is the one sprint goal?
  • What 1–3 tasks will get us there?
  • What does done look like for each?
  • Any blockers we should surface now?

Build (5–50 min)

  • Are you still on track for the goal?
  • Is scope growing? Cut something.
  • Blocked? Speak up now, not later.
  • Can you demo this yet?

Review (50–55 min)

  • Show me the working thing.
  • Does it match the sprint goal?
  • What checks did you run?
  • What feedback changes the next sprint?

Retro (55–60 min)

  • What went well?
  • What slowed you down?
  • One process change for next time?
  • What carries over and why?
Watch Out

Common Failure Modes

Too much scope
If you can't demo it in the last 10 min, you scoped too big.
Skipping review
No review = no verified output = wasted sprint.
Skipping retro
Without retro, you repeat the same mistakes.
Confusing activity with output
Lines of code ≠ shipped increment.
One person dominates
Both founders must own a deliverable.

Sprint Goal
One clear objective keeps two cofounders aligned. If it doesn't serve the goal, cut it.

          
Sprint Backlog
A tiny, visible list of what you committed to this cycle. Not a wishlist — a contract.

          
Definition of Done
A shared checklist for "finished." Prevents half-built features from piling up.

          
Increment
The usable output at the end. If you can't demo it, it didn't ship.

          
Retrospective
Five minutes of honest reflection. What worked, what didn't, what to try next.

          
Microsprint Loop
Plan (5 min) → Build (45 min) → Review (5 min) → Retro (5 min). Repeat every hour. Each person takes turns as Scrum Master for the other.
